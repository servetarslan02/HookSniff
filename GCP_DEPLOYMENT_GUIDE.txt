# HookSniff GCP Cloud Run Deployment — Setup Checklist
# ═══════════════════════════════════════════════════════════════════

## Prerequisites

- [x] Google Cloud account with billing enabled
- [x] `gcloud` CLI installed: https://cloud.google.com/sdk/docs/install
- [x] Docker installed: https://docs.docker.com/get-docker/
- [x] Git repository cloned locally

## Step 1: Google Cloud Project Setup

```bash
# 1a. Create GCP project (if not exists)
gcloud projects create hooksniff-app --name="HookSniff" --quiet

# 1b. Set as active project
gcloud config set project hooksniff-app

# 1c. Link billing account
# Go to: https://console.cloud.google.com/billing/projects
# Select your project and link a billing account (free tier available)

# 1d. Enable billing programmatically
gcloud services enable cloudresourcemanager.googleapis.com
```

## Step 2: Prepare Environment Configuration

```bash
# 2a. Copy environment template
cp deploy/env.production.example .env.production

# 2b. Edit .env.production with your actual values
# All [REQUIRED] fields MUST be filled:
#   - DATABASE_URL (from Neon)
#   - REDIS_URL (from Upstash)
#   - HMAC_SECRET (generate: openssl rand -hex 32)
#   - JWT_SECRET (generate: openssl rand -hex 32)
#   - POLAR_ACCESS_TOKEN (from Polar.sh)
#   - POLAR_WEBHOOK_SECRET (from Polar.sh)
#   - RESEND_API_KEY (from Resend)

nano .env.production
```

### Generating Secrets
```bash
# Generate HMAC_SECRET
openssl rand -hex 32

# Generate JWT_SECRET
openssl rand -hex 32

# Generate ENCRYPTION_KEY
openssl rand -hex 32
```

### Getting External Service Keys

#### Database (Neon - PostgreSQL)
1. Sign up: https://console.neon.tech
2. Create project
3. Get connection string: Copy full URL with password
4. Add to .env.production as DATABASE_URL

#### Cache (Upstash - Redis)
1. Sign up: https://console.upstash.com
2. Create Redis database
3. Copy REDIS_URL from Details tab
4. Add to .env.production

#### Payments (Polar.sh)
1. Sign up: https://polar.sh
2. Create organization
3. Get API token: https://dashboard.polar.sh/settings/tokens
4. Create products (Pro, Business)
5. Add to .env.production:
   - POLAR_ACCESS_TOKEN
   - POLAR_WEBHOOK_SECRET
   - POLAR_PRODUCT_PRO
   - POLAR_PRODUCT_BUSINESS

#### Email (Resend)
1. Sign up: https://resend.com
2. Get API key: https://resend.com/api-keys
3. Add to .env.production as RESEND_API_KEY

## Step 3: Authenticate with Google Cloud

```bash
# 3a. Authenticate with your Google account
gcloud auth login

# 3b. Set default project
gcloud config set project hooksniff-app

# 3c. Verify authentication
gcloud auth list  # Shows ACTIVE account
gcloud config get-value project  # Shows: hooksniff-app
```

## Step 4: Deploy to Google Cloud Run

```bash
# 4a. Make deploy script executable (Linux/Mac)
chmod +x deploy/gcp-deploy.sh

# 4b. Run deployment
./deploy/gcp-deploy.sh
```

### What the deployment script does:
1. ✅ Validates .env.production
2. ✅ Enables required GCP APIs
3. ✅ Creates Artifact Registry repository
4. ✅ Syncs secrets to Google Secret Manager
5. ✅ Builds Docker images locally
6. ✅ Pushes images to Artifact Registry
7. ✅ Deploys API, Worker, and Dashboard to Cloud Run
8. ✅ Prints service URLs

## Step 5: Verify Deployment

```bash
# 5a. Check service status
gcloud run services describe hooksniff-api --region=europe-west1

# 5b. Test API health endpoint
curl https://hooksniff-api-XXXXX.europe-west1.run.app/health

# 5c. View logs
gcloud run logs read hooksniff-api --region=europe-west1 --limit=50

# 5d. Run all diagnostics
bash deploy/troubleshoot-gcp.sh all
```

## Step 6: Configure Custom Domain (Optional)

```bash
# 6a. Add Cloud Run domain mapping
gcloud run domain-mappings create \
  --service=hooksniff-api \
  --domain=api.hooksniff.com \
  --region=europe-west1

# 6b. Update your DNS with Cloud Run's CNAME
# Go to your DNS provider and add:
# CNAME: api.hooksniff.com → ghs.googleusercontent.com

# 6c. Verify SSL certificate is provisioned
gcloud run domain-mappings describe api.hooksniff.com
```

## Common Issues & Solutions

### Issue: `.env.production` not found
```bash
cp deploy/env.production.example .env.production
# Then fill in all [REQUIRED] values
```

### Issue: Secret validation fails
```bash
# Check which [REQUIRED] fields are empty
grep "\[REQUIRED" .env.production

# Generate missing secrets
openssl rand -hex 32  # For HMAC_SECRET, JWT_SECRET, etc.
```

### Issue: Docker build fails
```bash
# Test local build
docker build -f Dockerfile.api .

# Check for missing dependencies
cargo check -p hooksniff-api
```

### Issue: API deployment fails (502 Bad Gateway)
```bash
# Check Cloud Run logs
gcloud run logs read hooksniff-api --limit=100

# Common causes:
# 1. Database connection failed → verify DATABASE_URL
# 2. Port mismatch → API must listen on PORT 3000
# 3. Health check failed → check /health endpoint

# Test locally
docker run -p 3000:3000 \
  -e DATABASE_URL="..." \
  -e REDIS_URL="..." \
  europe-west1-docker.pkg.dev/hooksniff-app/hooksniff/api:latest
```

### Issue: Secrets not loading
```bash
# Verify secrets in Secret Manager
gcloud secrets list

# Check secret value (first 100 chars)
gcloud secrets versions access latest --secret=hooksniff-db-url

# Ensure Cloud Run service account has secret access permission
# (Should be automatic, but verify if needed)
```

### Issue: Out of memory or timeout
```bash
# Increase memory
gcloud run deploy hooksniff-api \
  --memory=1Gi \
  --region=europe-west1

# Increase timeout (max 3600s)
gcloud run deploy hooksniff-api \
  --timeout=600 \
  --region=europe-west1
```
