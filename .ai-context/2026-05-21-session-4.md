### 1. Health Latency 0ms Bug Fix (20:35-20:50)
**Root cause:** PostgreSQL `AVG(INT)` returns `NUMERIC` type, which sqlx cannot deserialize into `f64`. The `unwrap_or_default()` silently swallowed the error, causing all latency values (avg, p95, p99) to show 0ms.

**Kanıt:** Node.js pg driver ile test — `AVG(INT)` string olarak dönüyor ("1169.333..."), `DOUBLE PRECISION` cast ile number oluyor.

**Fix:**
- `api/src/routes/health_endpoints.rs`: Cast `AVG()` and `percentile_cont()` to `::DOUBLE PRECISION`
- Replace all `unwrap_or_default()` with `match` + `warn!()` for proper error logging
- `dashboard/.../health/page.tsx`: Show "—" instead of "0ms" when no latency data
- `dashboard/src/schemas/api.ts`: Default latency fields to 0
