# React Query Migration - Batch 1

**Date:** 2026-05-16

## Summary

Converted 4 pages from `useState`+`useEffect` data fetching to React Query hooks. 1 page required no conversion.

## Pages Converted

### 1. `admin/email/page.tsx` — ⏭️ No conversion needed
- This page has **no data fetching** — only form state (`subject`, `body`, `planFilter`, etc.) and a mutation (`sendBulkEmail`)
- No `useEffect` or loading/error state for data loading
- Left unchanged

### 2. `admin/feature-flags/page.tsx` ✅
- **Before:** Manual `useState(flags, loading)` + `useCallback(fetchFlags)` + `useEffect`
- **After:** Uses existing `useAdminFeatureFlags()` hook for data, new mutation hooks for CUD operations
- **Hooks used:** `useAdminFeatureFlags`, `useCreateFeatureFlag`, `useUpdateFeatureFlag`, `useDeleteFeatureFlag`
- **Kept as useState:** `showCreate`, `editTarget`, `deleteTarget`, all form fields (`newName`, `editName`, etc.)
- **Removed:** `fetchFlags` callback, `loading` state, `creating`/`saving` states replaced by `mutation.isPending`
- **Type fix:** Used `FeatureFlagValidated` (from Zod schema) instead of `FeatureFlag` (from API types) since the hook returns schema-validated data with optional fields

### 3. `deliveries/[id]/page.tsx` ✅
- **Before:** Manual `useState(delivery, attempts, loading, error)` + `useCallback(fetchData)` + `useEffect`
- **After:** Uses new `useDeliveryDetail(id)` and `useDeliveryAttempts(id)` hooks
- **Hooks used:** `useDeliveryDetail`, `useDeliveryAttempts`, `useReplayWebhook`
- **Kept as useState:** `showReplayConfirm`, `copiedField`
- **Removed:** `fetchData` callback, manual `loading`/`error`/`delivery`/`attempts` state, `replaying` state replaced by `replayMutation.isPending`

### 4. `logs/page.tsx` ✅
- **Before:** Manual `useState(deliveries, total, loading, error, statusCounts)` + `useCallback(fetchData)` + `useEffect` + separate `setInterval` for auto-refresh
- **After:** Uses new `useDeliveryLogs` hook with `refetchInterval` parameter for auto-refresh
- **Hooks used:** `useDeliveryLogs`
- **Kept as useState:** `search`, `selected`, `attempts`, `attemptsLoading`, `autoRefresh` (controls `refetchInterval`)
- **Removed:** `fetchData` callback, `statusCounts` state (now from hook), manual `setInterval` (replaced by React Query `refetchInterval`)
- **Note:** `webhooksApi.getAttempts` still called inline on row click (event-driven, not mount-based fetching)

### 5. `notifications/page.tsx` ✅
- **Before:** Manual `useState(notifications, total, loading)` + `useCallback(fetchNotifications)` + `useEffect`
- **After:** Uses new `useNotifications` hook with query params
- **Hooks used:** `useNotifications`, `useMarkNotificationAsRead`, `useMarkAllNotificationsAsRead`, `useDeleteNotification`
- **Kept as useState:** `deleteTarget` (for confirm dialog)
- **Removed:** `fetchNotifications` callback, `loading`/`notifications`/`total` state
- **Bonus:** Mutation hooks include optimistic updates for mark-as-read and delete

## New Hooks Added

### `useAdminData.ts` (3 new hooks)
- `useCreateFeatureFlag()` — mutation, invalidates `['admin', 'feature-flags']`
- `useUpdateFeatureFlag()` — mutation, invalidates `['admin', 'feature-flags']`
- `useDeleteFeatureFlag()` — mutation, invalidates `['admin', 'feature-flags']`

### `useDashboardData.ts` (8 new hooks)
- `useDeliveryDetail(id)` — query for single delivery detail
- `useDeliveryAttempts(id)` — query for delivery attempts (catches errors, returns `[]`)
- `useDeliveryLogs(params)` — query combining list + 4 parallel status count fetches, supports `refetchInterval`
- `useNotifications(params)` — query for notification list with type/read filters
- `useMarkNotificationAsRead()` — mutation with optimistic update
- `useMarkAllNotificationsAsRead()` — mutation with optimistic update
- `useDeleteNotification()` — mutation with optimistic update
- `useReplayWebhook()` — mutation, invalidates delivery + logs + stats queries
