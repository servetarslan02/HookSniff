## ❌ KRİTİK (Çalışmayı Bozan)

### 1. Billing Portal — GET vs POST Method Mismatch
- **Frontend:** `GET /billing/portal` (api.ts:876 — default method)
- **Backend:** `POST /billing/portal` (billing.rs route: `.route("/portal", post(open_portal))`)
- **Etki:** 405 Method Not Allowed
- **Düzeltme:** Frontend `method: 'POST'` eklenmeli

### 2. Delivery Detail — Yanlış Endpoint Kullanılıyor
- **Frontend:** `webhooksApi.get()` → `GET /webhooks/{id}` → `DeliveryResponse` döner
- **Frontend bekler:** `DeliveryDetail` (request_headers, request_body, endpoint_url, error_message)
- **Backend:** `GET /webhooks/{id}/details` → `DeliveryDetails` (tüm alanlar mevcut)
- **Etki:** Delivery detail sayfasında request_headers, request_body, endpoint_url, error_message alanları boş görünür
- **Düzeltme:** Frontend `GET /webhooks/{id}/details` endpoint'ini kullanmalı

### 3. API Keys — `name` Alanı Eksik
- **Frontend:** `ApiKey.name: string | null`
- **Backend:** `ApiKeyInfo` struct'ında `name` yok, sadece `prefix` var
- **Etki:** API Keys tablosunda isim sütunu her zaman boş
- **Düzeltme:** Backend `ApiKeyInfo`'ya `name` alanı eklenmeli

### 4. Delivery Attempt — `status` Alan Uyumsuzluğu
- **Frontend:** `DeliveryAttempt.status: 'delivered' | 'failed'`
- **Backend:** `DeliveryAttempt.status_code: Option<i32>` (HTTP status kodu, delivery status değil)
- **Etki:** Status badge doğru gösterilmez
- **Düzeltme:** Backend'e `status` alanı eklenmeli veya frontend `status_code`'dan türetmeli

## ⚠️ ÖNEMLİ (Bozuk Alanlar)

### 5. Rate Limiting — Eksik Alanlar
- **Frontend bekler:** `endpoint_url`, `requests_per_minute`, `current_queue_depth`, `throttled_count`, `last_throttled_at`
- **Backend döner:** `endpoint_id`, `requests_per_second`, `burst_size`, `enabled`
- **Etki:** Çoğu sütun boş görünür
- **Düzeltme:** Backend response'a eksik alanlar eklenmeli

### 6. Billing Subscription — `current_period_end` Eksik
- **Frontend:** `BillingSubscription.current_period_end?: string`
- **Backend:** `SubscriptionResponse`'ta bu alan yok
- **Etki:** Sonraki fatura tarihi gösterilmez
- **Düzeltme:** Backend'e `current_period_end` eklenmeli

### 7. Portal Profile — `name` Eksik
- **Frontend:** `PortalProfile.name?: string`
- **Backend:** `ProfileResponse`'ta `name` yok
- **Etki:** Portal'da kullanıcı adı gösterilmez
- **Düzeltme:** Backend `ProfileResponse`'a `name` eklenmeli

### 8. Endpoint Response — `signing_secret` Eksik
- **Frontend:** `Endpoint.signing_secret?: string`
- **Backend:** `EndpointResponse`'ta `signing_secret` yok (güvenlik için kasıtlı çıkarılmış)
- **Etki:** Endpoint detayda signing secret gösterilmez
- **Düzeltme:** rotate-secret endpoint'i zaten secret döndürüyor, UI'da sadece rotate sonrası gösterilmeli

### 9. Admin System Health — Raw Fetch
- **Frontend:** `fetch(${API}/health)` — apiFetch yerine doğrudan fetch
- **Etki:** Auth refresh, retry, error handling yok. Ama `/health` public endpoint olduğu için çalışır
- **Düzeltme:** `apiFetch` kullanılmalı (düşük öncelik)
