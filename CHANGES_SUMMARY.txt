POST-PUSH INSTRUCTIONS FOR USER
═══════════════════════════════════════════════════════════════════

After pushing to GitHub:

1. Read the quick start:
   cat QUICK_START_GCP.txt

2. Sign up for external services:
   - Neon (PostgreSQL)
   - Upstash (Redis)
   - Polar.sh (Payments)
   - Resend (Email)

3. Create .env.production:
   cp deploy/env.production.example .env.production

4. Fill in all [REQUIRED] fields from external services

5. Deploy:
   chmod +x deploy/gcp-deploy.sh
   ./deploy/gcp-deploy.sh

6. Verify:
   bash deploy/troubleshoot-gcp.sh all

TESTING LOCALLY (OPTIONAL)
═══════════════════════════════════════════════════════════════════

Before deploying to GCP, test locally:

1. Build all images:
   docker build -f Dockerfile.api . -t hooksniff:api
   docker build -f Dockerfile.worker . -t hooksniff:worker
   docker build -f Dockerfile.dashboard . -t hooksniff:dashboard

2. Test with compose:
   docker compose -f docker-compose.prod.yml up

Note: First build takes 5-10 minutes due to Rust compilation.
      Subsequent builds are faster due to Docker caching.
